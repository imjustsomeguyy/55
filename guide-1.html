Saving Documents to the Shared Drive

How to save and organize files in the work shared drive correctly.

Shared Drive Basics
  1. Open the shared drive from the company portal or your file explorer's mapped network location.
  2. Find the correct top-level folder for your team or project — don't create a new top-level folder without checking with your manager first.
  3. Use the existing subfolder structure (e.g. Year > Project > Document Type) rather than dumping files in the root.
Naming & Versioning
  1. Name files clearly: YYYY-MM-DD_ProjectName_DocumentType_v1.ext
  2. Avoid "Final", "Final_Final", "Final_v2" — use version numbers instead and let the shared drive's built-in version history track older drafts.
  3. If the platform supports it (Google Drive, SharePoint, OneDrive), use "version history" rather than saving multiple copies of the same file.
Permissions & Sharing
  1. Check folder permissions before uploading anything sensitive — confirm who has access.
  2. Use "view" or "comment" access for documents that don't need to be edited by everyone.
  3. Don't move or delete other people's files without checking with them first.
Good Habits
  1. Save working files locally as a backup while editing, then sync/upload once finished.
  2. Archive completed projects into an "Archive" or "Completed" folder rather than leaving them mixed in with active work.
  3. Empty your own trash/recycle bin on the drive periodically if your org doesn't do it automatically — it counts against shared storage.
